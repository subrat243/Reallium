# Field Operative Guide

## Quick Start

### Installation

1. **Download the app** from your organization's app store
2. **Install** on your device (smartphone, body-cam, or tactical device)
3. **Launch** the Deepfake Detector app

### First-Time Setup

1. **Login** with your credentials
2. **Setup MFA** (Multi-Factor Authentication)
   - Scan QR code with authenticator app
   - Enter verification code
3. **Download models** (one-time, ~50MB)
4. **Grant permissions** (camera, microphone, storage)

## Using the App

### Analyzing Media

#### Live Camera/Microphone

1. Tap **"Live Detection"**
2. Point camera at subject or enable microphone
3. Wait for real-time analysis (2-3 seconds)
4. View results on screen

#### Existing Files

1. Tap **"Analyze File"**
2. Select video or audio file
3. Wait for analysis
4. View detailed results

### Understanding Results

#### Authenticity Score

- **Green (0.7-1.0)**: Likely authentic
- **Yellow (0.3-0.7)**: Uncertain, review carefully
- **Red (0.0-0.3)**: Likely deepfake

#### Confidence Level

- **High (>0.8)**: Trust the result
- **Medium (0.5-0.8)**: Use caution
- **Low (<0.5)**: Manual review recommended

### Offline Mode

The app works **completely offline** - no internet required!

1. Ensure models are downloaded
2. Enable **"Offline Mode"** in settings
3. Analyze media as normal
4. Results sync when online

### Viewing History

1. Tap **"History"**
2. View past detections
3. Filter by date, type, or result
4. Export reports

## Field Operations

### Best Practices

1. **Verify Source**
   - Always verify media source
   - Check metadata when possible
   - Cross-reference with other intelligence

2. **Multiple Checks**
   - Run analysis multiple times
   - Use different devices if available
   - Compare results

3. **Document Everything**
   - Take screenshots of results
   - Note environmental conditions
   - Record chain of custody

### Common Scenarios

#### Interrogation Recording

1. Record audio/video during interrogation
2. Analyze immediately after
3. Flag any suspicious segments
4. Document in report

#### Social Media Content

1. Download media from platform
2. Analyze with app
3. Check multiple versions if available
4. Report findings to command

#### Field Evidence

1. Capture media at scene
2. Analyze on-site
3. Secure original files
4. Sync results when safe

## Troubleshooting

### App Won't Start

- Restart device
- Reinstall app
- Check device compatibility

### Analysis Failed

- Check file format (MP4, AVI, WAV, MP3)
- Verify file isn't corrupted
- Ensure sufficient storage

### Slow Performance

- Close other apps
- Clear app cache
- Update to latest version

### Models Not Downloading

- Check internet connection
- Verify storage space (need 100MB free)
- Try manual download in settings

## Security

### Device Security

1. **Lock Screen**: Always use PIN/biometric
2. **Encryption**: Enable device encryption
3. **Updates**: Keep app and OS updated
4. **Lost Device**: Report immediately

### Data Security

1. **Sensitive Media**: Mark as classified
2. **Sharing**: Only via secure channels
3. **Deletion**: Use secure delete feature
4. **Backup**: Follow organizational policy

### Reporting Threats

If you detect a deepfake:

1. **Document**: Screenshot results
2. **Preserve**: Save original media
3. **Report**: Follow chain of command
4. **Secure**: Encrypt if classified

## Advanced Features

### Batch Processing

1. Select multiple files
2. Tap **"Batch Analyze"**
3. Wait for all results
4. Export batch report

### Explainability View

1. After analysis, tap **"Explain"**
2. View attention heatmaps
3. See which regions triggered detection
4. Use for evidence presentation

### Custom Thresholds

1. Go to **Settings**
2. Tap **"Detection Threshold"**
3. Adjust slider (default: 0.5)
4. Higher = fewer false positives

## Appendix

### Supported Formats

**Video**: MP4, AVI, MOV, MKV
**Audio**: WAV, MP3, AAC, FLAC

### System Requirements

**Android**: 8.0+ (2GB RAM minimum)
**iOS**: 12.0+ (iPhone 7 or newer)

### Battery Life

- **Live Detection**: ~2 hours continuous
- **File Analysis**: ~50 analyses per charge
- **Standby**: 24+ hours

### Storage Requirements

- **App**: 150MB
- **Models**: 50MB
- **Cache**: 100MB (configurable)
